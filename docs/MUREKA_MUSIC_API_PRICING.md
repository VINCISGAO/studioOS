## 1. 计费模型（先理解这个）

Mureka API 采用 **「预充值档位 → 并发上限 + 按次单价」** 双层计费：

1. **充值档位（Plan）** — 决定账号 **最大并发请求数**，余额 **12 个月有效**
2. **按次单价** — 每次 API 调用按「首歌 / 条歌词 / 个 Vocal ID / 小时」等单独计费，从余额扣减

```
总成本 ≈ 充值档位（买并发） + Σ(各 API 调用次数 × 单价)
```

### 1.1 标准充值档位（全站通用并发阶梯）

| 档位名称 | 充值金额 | 最大并发 | 备注 |
|----------|----------|----------|------|
| 踪迹 Traces | $30 | 1 | 含完整访问；入门档 |
| 基本 Basic | $1,000 | 5 | 完整 API 访问 |
| 标准 Standard | $3,000 | 15 | 完整 API 访问 |
| 商业 Business | $5,000 | 25 | 完整 API 访问 |
| 企业 Enterprise | $30,000 | 150 | 完整 API 访问 |
| 定制 | 联系销售 | 可谈 | 免费试用申请、批量折扣、专属技术顾问 |

**注意：**

- $30 档位写的是「完全访问」，$1,000+ 写的是「完整 **API** 访问」——接入前建议向销售确认 Canvas 所需接口是否都在 API 范围内。
- 部分能力（Stem 分离、音乐转录、TTS）默认仅 **1 并发**，提高并发需 **联系销售**。
- 并发是 **账号级** 上限，不是单次请求生成数量；单次 batch 仍可一次生成最多 **3 首**（默认 2 首）。

## 3. 各 API 详细说明

### 3.1 歌曲 / 歌词（Song & Lyrics）

**适用：** 标准 AI 写歌 + 歌词，品牌/创作者快速出 demo。

| 模型 | 单价 | 时长上限 | 单次 batch |
|------|------|----------|------------|
| Oxygen / V8 / V9 | $0.045 / 首 | 5 分 30 秒 | 最多 3 首（默认 2） |
| V7.6 | $0.03 / 首 | 5 分 30 秒 | 同上 |

**能力：**

- 文本描述音乐风格
- 参考曲目灵感创作
- 指定人声演唱
- 录制旋律灵感启动写歌
- 支持完整歌曲下载，默认生成歌词

**并发：** 遵循标准充值档位（$30→1 … $30,000→150）。

---

### 3.2 提示唱歌（Prompted Singing）

**适用：** 给定风格 + 文本 prompt 生成 **带歌词的完整歌曲**（比 3.1 更「定向」）。

| 项目 | 值 |
|------|-----|
| 单价 | **$0.30 / 首** |
| 模型 | V7.6、Oxygen、V8、V9 |
| 时长 | 最长 5 分 30 秒 |
| batch | 最多 3 首 |

**并发：** 标准档位。

**成本提示：** 约为 V7.6 歌曲生成的 **10 倍**，适合高价值场景，不适合 bulk demo。

## 5. 成本估算示例

### 5.1 Invite-only Beta（轻量）

假设：10 品牌 × 每人 5 次 BGM，50 创作者 × 每人 3 次 demo 歌

| 项目 | 次数 | 单价 | 小计 |
|------|------|------|------|
| BGM (V7.6) | 50 | $0.03 | $1.50 |
| 歌曲 (V7.6) | 150 | $0.03 | $4.50 |
| **API 按次合计** | | | **~$6** |
| **最低充值（并发）** | | | **$30**（1 并发，12 个月有效） |

Beta 阶段 **$30 踪迹档** 足够；按次成本可忽略，瓶颈在并发 1。

### 5.2 小规模上线（需 5 并发）

| 项目 | 假设 | 月 API 按次 | 月按次成本 |
|------|------|-------------|------------|
| BGM | 2,000 次 | V7.6 | $60 |
| 歌曲 | 1,000 次 | V7.6 | $30 |
| 歌词 | 3,000 套 | — | $27 |
| **合计** | | | **~$117 / 月** |

需 **$1,000 基本档**（5 并发）+ 余额续充（按实际调用扣费）。

### 5.3 单次用户操作 → Credits 换算参考

若 VINCIS 以 Credits 售卖，可参考 **V7.6 BGM $0.03** 为基准：

```
1 次 BGM ≈ $0.03
若 1 Credit = $0.01 → 单次 BGM ≈ 3 Credits（未含毛利）
若平台抽成 30% → 对用户收 ≈ 4–5 Credits
```

具体 Credits 定价需对齐 `docs/PRODUCTION_PRICING_ENGINE.md`，此处仅作 API 成本底数。

## 6. 接入 VINCIS 时的工程注意点

1. **Worker 并发** — 与 Mureka 账号并发上限对齐；Beta 仅 1 并发时需 **队列 + 轮询**，避免 burst 被拒。
2. **幂等与退款** — 生成失败是否扣费以 Mureka 账单为准；VINCIS Credits 释放逻辑应对齐 `canvas-music-generation.service` 现有模式。
3. **存储** — 输出音频进 R2；Stem 的 .midi 需单独存储策略。
4. **语言** — 延长/混音等仅保证英/西/葡/中/日/韩；Canvas 多语言 UI 不等于生成语言支持。
5. **定制模型** — 品牌包长期合作再谈；Beta 不建议上 Custom Model 训练成本。
6. **环境变量** — API Key 放 Vercel/GitHub Secrets，与 Deposit 配置方式相同，勿进 `NEXT_PUBLIC_*`。

### 6.1 VINCIS 已接入（2026-07-23）

| 项目 | 说明 |
|------|------|
| 环境变量 | `MUREKA_API_KEY`（可选 `MUREKA_API_BASE_URL`，默认 `https://api.mureka.ai`） |
| 状态检查 | `GET /api/v1/mureka/status` |
| 生成入口 | Canvas `POST /api/generation/music` → `canvas-music-generation.service` |
| 客户端 | `lib/canvas/mureka-client.ts` |
| 模式路由 | 器乐 → `/v1/instrumental/generate`；简单 → `/v1/song/easy-generate`；自定义人声 → 歌词（可选）+ `/v1/song/generate` |
| 轮询 | song / instrumental query，最长 8 分钟；`n=1` 避免默认双份计费 |
| 模型映射 | `v7.5-basic` → `mureka-7.6`；studio → `mureka-8`；其余 → `auto` |
